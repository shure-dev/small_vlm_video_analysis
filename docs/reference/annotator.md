# Video Annotationアプリ

`sop-app` は、動画区間の人手アノテーションと推論結果レビューをまとめた動画専用Webアプリです。

## 起動

```bash
sop-app --dataset factory_ego
sop-app --dataset factory_ego --unit f001_w004_material_replenishment
sop-app --port 9000 --no-browser
sop-view --dataset factory_ego  # 読み取り専用
```

## 基本操作

1. サイドバーのデータセットと動画から対象を選ぶ
2. 左側で動画を一度通して見て、「確認位置」を動作の境界付近へ合わせる
3. 右側の「イベント」で「追加」を選ぶ
4. 映像で直接観察できる動作を日本語で入力し、発生区間を選んで追加する
5. 既存イベントはタブから切り替え、数値入力またはタイムラインで境界を直す
6. 同じ動作が再度起きたら「現在位置から区間を追加」で同じイベントへ区間を増やす
7. 厳密な境界は「静止画で確認」と1フレーム移動を使って確認する
8. 候補イベントが存在しない場合だけ「起きていない（該当なし）」を選ぶ
9. 下部のタイムラインで動画全体の区間を見直す

イベント文と区間は変更時に原子的に自動保存されるため、別の動画へ移動して後から再開できます。進捗や承認状態は保存せず、アノテーションの事実だけを残します。

## 記述ルール

- 映像で直接確認できる動作を書く
- 1イベントに1動作を書く
- 「作業者が部品を箱に入れる」のような短い現在形にする
- 手段が識別に重要なら省略しない（袋を逆さにして落としているなら、単に「取り出す」と書かない）
- 意図、原因、結果を推測しない
- 同じ動作が繰り返される場合は、イベントを複製せず同じイベントへ複数区間を付ける
- 候補イベントを確認したが起きなかった場合だけ「起きていない」を使う

未入力と「起きていない」は異なります。候補イベントが映像内にないことを確認した場合だけ `null` として保存します。

## 保存先

イベント文は推論用SOPとannotation JSONの `event_labels` の両方へ同期し、区間は `events` に保存されます。JSON単体でもイベントの意味と区間を読めます。保存先は `datasets/<dataset>/annotations/human/<unit>.json` で、書き込みは一時ファイルからの原子的置換です。

```json
{
  "unit_id": "clip_001",
  "annotation_revision": "human",
  "interval_convention": "half-open_seconds",
  "event_labels": {
    "event_001": "作業者が部品を箱に入れる"
  },
  "events": {
    "event_001": [
      {"start_s": 2.5, "end_s": 5.0},
      {"start_s": 8.0, "end_s": 9.5}
    ]
  }
}
```

## 結果レビュー

「結果レビュー」は、`runs/<run-id>/predictions/` にhumanイベントIDと一致するpredictionがある場合に利用できます。人手区間とモデル区間を同じ時間軸に表示し、unit単位のmean tIoUとtIoU@0.5を示します。イベント表はtIoUの低い順なので、改善すべきイベントから確認できます。推論自体はアプリ内では実行しません。
